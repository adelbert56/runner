# 第二大腦索引

這裡整理跑者廣場目前最重要的實作經驗與維護入口。後續代理人接手時，先看根目錄 `SKILLS.md`，再依任務讀下列文件。

## 核心文件

- `../../SKILLS.md`：專案級代理人準則，包含資料、UI、菜單、驗證與自我迭代規則。
- `资料更新SOP.md`：自動排程、資料流程、人工待補與發布檢查。
- `../赛事/平台爬虫覆盖报告.md`：官方報名平台命中、更新與解析錯誤。
- `../赛事/资料品质报告.md`：目前資料缺漏與優先補資料方向。
- `../赛事/爬虫追踪计划.md`：依開報日、截止日與賽事日期安排下一次追蹤。

## 目前已沉澱的經驗

- UI 不要一直加資訊，要把資訊分層：卡片顯示必要判斷，來源、場地、費用、名額放展開區或報告。
- 手機版要實際開 viewport 看，不能只靠 CSS 推測。
- 賽事報名入口要盡量直連官方平台，避免導到強迫登入的中介頁。
- 報名日期已開始但抓不到資料，通常不是資料不存在，而是平台解析器還不夠完整。
- 開報後待補資料應該是優先修 parser 的清單，不只是人工補資料清單。
- 跑鞋與新聞需要定期補候選資料，否則分頁會像空功能。
- 練跑菜單是商品化亮點，重點是依目標賽事日期、週數、跑量與配速自動調整。

## 下一輪優先順序

1. 針對開報後待補清單修平台 parser，優先補地點、主辦、費用、名額。
2. 補跑鞋與新聞候選來源，保留日期、來源、分類、收藏與排序。
3. 繼續手機版實測，避免卡片操作區、篩選區、菜單表單溢出。
4. 把常見錯誤回寫到 `SKILLS.md`，讓後續代理人直接避開。
