# 內容來源與爬蟲規劃

更新日：2026-05-15

這份清單用來管理跑鞋新品、評測心得、跑步新聞與訓練知識的來源。目標不是把所有文章都搬進網站，而是由爬蟲建立候選池，再由規則自動挑出對跑者有決策價值的內容發布。

## 目前沒有專用 MCP

目前沒有「跑鞋新聞專用 MCP」。可用流程是：

- 透過來源爬蟲定期找中文跑鞋與跑步內容。
- 把來源、日期、分類、分數、摘要與適合跑者整理進候選池。
- 前端只發布精簡後的卡片，避免變成大量新聞轉貼。
- 定期爬蟲會產生候選清單並自動上架；候選不足時使用上一版已發布內容補位，避免前台商品突然變少。

## 候選內容欄位

每筆候選內容至少記錄：

- 日期
- 標題
- 來源網站
- 原文連結
- 分類：跑鞋新品、跑鞋評測、訓練、恢復、補給、賽事觀念
- 適合跑者：新手、10K、半馬、全馬、越野、競速
- 跑者決策重點
- 是否上架
- 查證時間

## 定期爬蟲節奏

- 每週一：跑鞋新品與評測來源更新。
- 每週三：訓練、恢復、補給與跑步新聞更新。
- 每次上架後：執行 `npm run content:quality:strict`，擋下數量不足、重複 URL、摘要缺漏或低資訊量內容。
- 每月一次：檢查上一版庫存補位比例，若補位過高就新增來源或調整關鍵字。

## 上架原則

- 優先中文內容。
- 不收純廣告稿，除非能整理出明確用途與適合跑者。
- 每張卡片必須回答「這對跑者有什麼用」。
- 跑鞋內容不要只寫科技名詞，要寫適合課表與不適合情境。
- 新聞內容要能連到練跑、補給、恢復或賽事決策。
